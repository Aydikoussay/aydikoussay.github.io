Hack The Box — Windows Medium

Principal Writeup

Author: atrox | Tags: Active Directory, Kerberos, AS-REP Roasting, SMB Enumeration

Overview

Principal is a Windows Active Directory machine focusing on Kerberos exploitation. The path involves discovering user lists via SMB, performing AS-REP Roasting to obtain credentials, and exploiting domain object permissions to reach Domain Admin.

Reconnaissance

Nmap

nmap -sC -sV -oA nmap/principal 10.10.11.231

Standard Active Directory port profile identified:

PortService
53DNS
88Kerberos
135, 445RPC / SMB
389, 3268LDAP / Global Catalog
5985WinRM

Target Domain: principal.htb

Enumeration

SMB Share Access

Initial reconnaissance shows anonymous access allowed on certain shares. Exploring these reveals a users.txt file, which is a key piece of information for user-based attacks.

smbclient -L //10.10.11.231/ -N

Inside the Public share, we find a users.txt file. This list is essential for performing Kerberos-based attacks without needing prior passwords.

Initial Foothold

AS-REP Roasting

Checking for users with Kerberos pre-authentication disabled (DONT_REQ_PREAUTH):

impacket-GetNPUsers principal.htb/ -usersfile users.txt -format hashcat -outputfile hashes.asrep

The user m.rossi is vulnerable. Cracking the hash with hashcat -m 18200 reveals the password.

WinRM Shell

evil-winrm -i 10.10.11.231 -u m.rossi -p 'password123'

Privilege Escalation

BloodHound Analysis

Using bloodhound-python, we map out the domain relationships. We identify that m.rossi has specific write permissions over sensitive objects.

python3 bloodhound.py -u 'm.rossi' -p '...' -d principal.htb -c All

GMSA Exploitation

The escalation path involves Group Managed Service Accounts (GMSA). By exploiting permissions, we retrieve the GMSA password to impersonate a domain administrator.

Attack Chain Summary

SMB Enumeration (users.txt) → AS-REP Roasting → Hashcat → WinRM Shell → BloodHound → GMSA Exploitation → Domain Admin

Tools Used

nmap smbclient impacket hashcat BloodHound evil-winrm
