Hack The Box — Windows Medium

Principal Writeup

Author: atrox | Tags: Active Directory, Kerberos, AS-REP Roasting, SMB Enumeration

Overview

Principal is a medium-difficulty Windows machine that focuses on Active Directory exploitation. The attack path involves enumerating SMB shares to find a list of users, performing AS-REP Roasting to gain an initial foothold, and leveraging misconfigurations within the domain to escalate privileges.

Reconnaissance

Nmap Scan

nmap -sC -sV -oA nmap/principal 10.10.11.231
PortService
53DNS
88Kerberos
135MSRPC
389LDAP
445SMB

Enumeration

SMB Share Access

Initial reconnaissance shows anonymous access allowed on certain shares. Exploring these reveals a users.txt file, which is a key piece of information for user-based attacks.

smbclient -L //10.10.11.231/ -N

Inside the Public share, we find a list of potential domain users. This list is essential for performing Kerberos-based attacks without needing prior passwords.

Initial Foothold

AS-REP Roasting

With the list of usernames, we can check for accounts that do not require Kerberos pre-authentication. This allows us to request an authentication ticket and attempt to crack it offline.

impacket-GetNPUsers principal.htb/ -usersfile users.txt -format hashcat -outputfile hashes.asrep

The tool identifies that the user m.rossi has pre-authentication disabled. Using hashcat with mode 18200, we successfully crack the hash to reveal the password.

WinRM Access

With valid credentials for m.rossi, we authenticate via WinRM to establish our initial foothold on the system.

evil-winrm -i 10.10.11.231 -u m.rossi -p 'password123'

Privilege Escalation

Active Directory Enumeration

Running BloodHound-python from our attack machine helps visualize the attack path within the domain. We identify that our current user has GenericWrite or WriteProperty permissions over other sensitive objects.

python3 bloodhound.py -u 'm.rossi' -p '...' -d principal.htb -c All

Path to Domain Admin

The enumeration reveals that we can exploit the relationship between service accounts or GMSA (Group Managed Service Accounts). By leveraging these misconfigurations, we can impersonate a high-privileged user and eventually gain Domain Admin privileges.

Attack Chain: SMB (users.txt) → AS-REP Roasting → Hashcat → WinRM Foothold → AD Enumeration → Domain Admin
